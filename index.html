⚙️ Konfigurasi EDC Connection

MODE API MIDDLEWARE

Transaksi dikirim melalui middleware API ke EDC via Ably realtime messaging. Tidak perlu koneksi langsung ke EDC dari browser.

Panduan Setup:
  1. Isi API Base URL: https://development-ecrlink.pcsindonesia.com
  2. Isi MID (Merchant ID) dan TID (Terminal ID) yang sudah terdaftar di middleware
  3. Klik Test Connection untuk memastikan middleware aktif
  4. Lakukan transaksi — POS akan mengirim request ke middleware, lalu middleware meneruskan ke EDC via Ably
⚠️ Catatan Penting:
  • MID/TID harus sudah terdaftar di middleware sebelum bisa mengirim transaksi
  • Timeout default: 60 detik — jika EDC tidak merespon, middleware mengembalikan error 504
  • Token transaksi dienkripsi otomatis menggunakan AES-128-ECB (sama seperti mode WebSocket)
Flow Transaksi:

POS → POST /api/v1/transaction → Middleware → Ably → EDC → Ably → Middleware → Response ke POS

Masukkan IP address EDC device
Default: 6745 (WS), 6746 (WSS) - Sesuaikan dengan ECR Link
Base URL middleware API (default: https://development-ecrlink.pcsindonesia.com)

Secret key untuk AES/ECB/PKCS5Padding (default: ECR2022secretKey)
